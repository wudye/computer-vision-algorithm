交互式图像分割（Interactive Image Segmentation）的基本定义和分类。 
交互式图像分割是一种计算机视觉技术，它不依赖算法全自动处理，而是通过引入人类的先验知识（即“交互”）来引导算法更准确地识别目标物体。 
## 1. 交互式图像分割的步骤

* 人工标注：用户首先在图像上进行简单的操作（如画线、点选或框选），告诉算法哪些像素属于“前景”（感兴趣的物体），哪些属于“背景”。
* 算法分类：算法（如 [GrabCut](https://patents.google.com/patent/CN103559719A/zh) 或 Graph Cut）根据这些已知点提取颜色、纹理或位置特征，并自动推断剩余像素的类别。
* 得到结果：最终将图像划分为若干个有意义的子区域（通常是前景目标和背景）。 [ 

## 2. 常见的交互类型
根据用户的操作方式，主要分为以下几种：

* 画笔 (Scribbles)：用户在目标上涂抹几笔，算法根据笔划覆盖的像素颜色进行区域生长或图割。
* 画框 (Bounding Box)：最典型的例子是 [GrabCut 算法](https://blog.csdn.net/weixin_29044713/article/details/158111303)。用户画一个矩形框住目标，框外强制设为背景，框内则通过算法区分前景。
* 画点 (Points)：在精细分割或深度学习模型（如 [Segment Anything Model (SAM)](https://fuxi.163.com/database/1121)）中常用。用户通过点击几个关键点来指定目标位置。 

这种技术广泛应用于照片抠图、医学影像标注和自动驾驶数据预处理等领域。




在图像分割（特别是基于图割（Graph Cut）的算法）中，“容量”（Capacity）是一个核心概念。它代表了图中边（Edge）的权重，用于衡量将两个相邻像素点划分到不同区域（如前景和背景）所需要付出的“代价”。 [1, 2] 
根据边的类型，容量通常分为以下两类定义：
## 1. 区域项容量（T-links / Terminal Links）
这种边连接像素点与两个特殊的终端节点：源点（Source, $s$）和汇点（Sink, $t$）。 [2, 3] 

* 定义：反映了某个像素属于特定类别（如前景或背景）的概率或相似度。
* 计算方式：通常基于像素的颜色、灰度或位置统计。例如，如果一个像素点的颜色非常符合前景的概率模型，那么它与 $s$ 点连接的边容量就会很大。 [3, 4] 

## 2. 边界项容量（N-links / Neighbor Links）
这种边连接图像中相邻的像素点。 [1] 

* 定义：反映了相邻像素之间的不连续性或差异度。
* 计算方式：通常与像素间的灰度/颜色差异成反比。
* 差异大：如果两个像素颜色差别很大，说明它们之间很可能是物体的边缘，此时边的容量（权重）就很小，算法更容易从这里“切断”。
   * 差异小：如果两个像素颜色接近，容量就很大，意味着切断它们的代价很高，算法倾向于将它们保留在同一区域内。 [1, 3] 

## 容量与分割的关系
在算法执行时，它会寻找一个最小割（Min-Cut），即寻找一组边，使得这些边的容量总和最小，并将图分为不相连的两部分。 [5, 6] 

* 最小割 = 最大流：根据最大流最小割定理，最小割的容量和等于从源点到汇点的最大流量。
* 物理意义：最小化总容量意味着在“最像边缘的地方”进行切割，同时最贴合预设的前背景概率模型。 [3, 4, 7]
